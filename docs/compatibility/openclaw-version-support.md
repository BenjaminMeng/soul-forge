# OpenClaw 版本兼容性

## Soul Forge v3.1.x

| OpenClaw 版本 | 兼容性 | 备注 |
|---------------|--------|------|
| 1.4.x+ | ✅ 完全兼容 | 推荐版本 |
| 1.3.x | ✅ 兼容 | 需要手动启用 hooks |
| 1.2.x | ⚠️ 部分兼容 | Bootstrap Hook 正常，但无 `workspace.dir` 自动注入 |
| < 1.2.0 | ❌ 不支持 | Skill 加载机制不兼容 |

> **推荐：** 使用 OpenClaw 1.4.x 或更高版本以获得完整功能。

## 运行时要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Node.js | 18.x | 22.x LTS |
| OpenClaw | 1.2.0 | 1.4.x+ |
| OS | Windows 10 / macOS 12 / Ubuntu 20.04 | 最新稳定版 |

## 已测试的 AI 模型

以下模型完成了 Soul Forge 的初步发布前验证（单元测试 + 端到端流程）：

| 模型 | 状态 | 备注 |
|------|------|------|
| DeepSeek V3 | ✅ 初步验证通过 | 单元测试 + E2E 端到端 |
| MiniMax M2.1 | ✅ 初步验证通过 | 单元测试 + 半自动端到端 |

> **注意：** 跨模型系统测试（更多模型覆盖）仍在进行中，结果将持续更新。
> 现有测试覆盖核心安装流程和主要功能，但不代表所有边缘场景均已验证。
> MANDATORY 指令的跟随程度因模型而异。

## 已知限制

- **MiniMax M2.1**：`telemetry_status` 和 `telemetry_disable` 的回复措辞可能不严格遵循 SKILL.md 模板（DEF-3/DEF-4，已接受为模型限制）
- **所有模型**：首次运行时，`answers_hash` 可能不被写入 config（DEF-2，SKILL.md 修复中）
- **语言检测**：英文优先规则在部分模型上跳过（DEF-1，SKILL.md 修复中）
